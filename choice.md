# 实验设计与分析初稿

## 1. 目标与概述
- **1.1 核心问题**：怎样才能高效地评估并筛选出高质量的训练数据，从而提升监督学习模型——尤其是大语言模型（LLM）的微调效果？
- **1.2 实验思路**：我们利用 KNN-Shapley 框架进行数据估值。实验设计覆盖了图像分类（CNN）和指令微调（LLM）两个任务，目的是为了验证数据清洗是否真的有效，并弄清楚不同的特征表示方式（图像特征、文本梯度、语义表示）对最终的估值效果到底有什么影响。

## 2. 实验一：图像分类任务上的验证与改进
*目的：先在相对简单的 CV 任务上跑通 KNN-Shapley，验证它的有效性并寻找优化空间。*

### 2.1 设置
- **数据集**：Cats vs. Dogs。我们构建了一个含噪声版本，随机翻转了部分标签。
- **基线**：原始的 Soft-label KNN-SV 算法，使用欧氏距离。
- **指标**：AUROC, F1-Rank (Top-10%), F1-Cluster。
- **改进点**：我们对特征向量做了 **L2 归一化**。这样做是为了让欧氏距离等价于余弦距离，从而更精准地衡量特征在方向上的相似度。

### 2.2 特征提取器对比
- **方案**：保持 KNN-Shapley 算法不变，轮换不同的 Backbone 提取特征，包括 VGG11, ResNet-18/50, MobileNetV2, DenseNet121, EfficientNet-B0, ConvNeXt-Tiny, ViT-B/16 等。
- **发现**：EfficientNet-B0 的综合指标最好。这说明轻量级的高效网络在提取特征方面优势很明显。

### 2.3 相似度度量对比（核心贡献一）
- **方案**：基于表现最好的 EfficientNet-B0，我们对比了**欧氏距离**和**余弦距离**的效果。
- **结果**：
  | 模型/距离           | AUROC | F1-Rank   | F1-Cluster |
  | ------------------- | ----- | --------- | ---------- |
  | EfficientNet (欧氏) | 0.995 | **0.910** | 0.261      |
  | EfficientNet (余弦) | 0.991 | 0.845     | **0.354**  |
  - **分析**：虽然余弦距离在 F1-Rank 上稍有下降，但 **F1-Cluster 指标提升了 35% 之多**。这意味着在特征空间里，余弦相似度能把脏数据和干净数据分得更开，更有利于聚类算法自动识别出噪声。

### 2.4 清洗效果验证
- **方案**：
  1. 先用噪声数据训练一个逻辑回归分类器（`Dirty Model`）。
  2. 使用 KNN-Shapley（配合余弦距离）筛选掉低价值数据。
  3. 用清洗后的数据重新训练分类器（`Clean Model`）。
- **结果**：`Clean Model` 在干净验证集上的准确率明显压过了 `Dirty Model`。
- **结论**：至少在图像分类上，这套清洗逻辑是站得住脚的，确实能提升下游模型的性能。

## 3. 实验二：LLM 监督微调中的数据估值迁移
*目的：把这套方法搬到更复杂的 LLM SFT 任务上，看看能不能搞定文本数据估值，顺便对比下不同的特征表示方案。*

### 3.1 设置
- **模型**：Qwen1.5-0.5B。
- **数据集**：Alpaca 指令集。我们注入了 30% 的噪声，操作是把原本的回答替换成 "I don't know" 这种无意义文本。
- **指标**：训练 Loss、ROUGE-L。
- **基线对比**：
  - `Dirty Model`：直接硬吃全量噪声数据。
  - `Oracle Model`：用全量干净数据训练（这是理论天花板）。
  - `Gradient-KNN Model`：用梯度相似度 + KNN-Shapley 清洗。
  - `RepSim-Mean Model`：用语义表示 + 平均相似度清洗。
  - `RepSim-KNN Model`：用语义表示 + KNN-Shapley 清洗（这是我们的主打方案）。

### 3.2 方法对比（核心贡献二）
- **方案**：我们排列组合了两种特征（梯度相似度 vs 语义表示 RepSim）和两种聚合方式（平均聚合 vs KNN-Shapley），用它们清洗数据后对 Qwen 跑 SFT。
- **核心结果**（示例占位）：
  | 模型                         | 训练数据量 | Avg ROUGE-L | 排名 |
  | ---------------------------- | ---------- | ----------- | ---- |
  | Oracle Model (天花板)        | 1000       | 0.3306      | 1    |
  | **Clean Model (RepSim-KNN)** | 700        | **0.2889**  | 2    |
  | Clean Model (Gradient-KNN)   | 700        | 0.2636      | 3    |
  | Clean Model (RepSim-Mean)    | 700        | 0.2603      | 4    |
  | Dirty Model (底线)           | 1000       | 0.2314      | 5    |

- **分析**：
  1. **KNN 很管用**：`RepSim-KNN` 明显比 `RepSim-Mean` 强，这说明评估邻域贡献度比单纯看全局平均相似度要准得多。
  2. **特征选对了**：`RepSim-KNN` 赢了 `Gradient-KNN`。在 SFT 任务里，**模型最后一层的语义表示**比参数梯度更能分清好数据和坏数据，而且算起来还快。
  3. **洗洗更健康**：所有的清洗模型（哪怕只用了 70% 的数据）都比硬吃 100% 噪声数据的 `Dirty Model` 强。这也证明了筛选是有必要的。目前看 `RepSim-KNN` 是最接近 Oracle 天花板的方案。

## 4. 总结与待填坑
- **做完的**：
  1. CV 任务那边，验证了 KNN-Shapley 没问题，而且我们改用的**余弦相似度**确实让聚类分得更开了。
  2. LLM SFT 这边，成功把框架迁过来了，五个模型的对照实验也跑通了。
  3. 提出了 **RepSim-KNN**，数据证明它处理语义噪声比梯度方案强，目前它是最优解。
- **还得补的（为了回应老师说的“结论要硬”）**：
